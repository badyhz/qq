# AGENT_RULES

你是本项目的执行型开发代理。必须遵守以下规则：

## 1. 开发原则
- 不要擅自改变项目目标
- 不要删除已有文件，除非任务明确要求
- 不要跳过测试与验收
- 不要一次处理多个大任务
- 优先做“可运行、可验证、可回滚”的实现

## 2. 工作顺序
每次开始工作前，必须先阅读：
1. PROJECT_STATE.md
2. TASKS.md
3. acceptance.json
4. docs/ARCHITECTURE.md

## 3. 执行要求
- 一次只处理一个任务
- 每完成一个任务，都要说明：
  - 改了哪些文件
  - 为什么这样改
  - 如何验证
  - 当前风险点是什么
- 新代码必须尽量模块化
- 关键逻辑必须写必要注释
- 涉及交易、下单、风控时，默认先实现 dry-run
- 未经明确许可，不得直接启用真实下单

## 4. 安全规则
- 不得在代码中硬编码 API Key / Secret
- 所有密钥只能从环境变量读取
- 默认保守风控
- 涉及真实交易功能时，必须设置显式开关

## 5. 输出要求
每次完成任务后，输出格式必须包含：
1. 本次完成内容
2. 修改文件列表
3. 运行方式
4. 验收结果
5. 下一步建议
