# Human Approval Timestamp Policy

**Task ID:** T1294
**release_hold:** HOLD

## Purpose

Defines timestamp requirements for human approval evidence packs.
Ensures temporal integrity and prevents backdating.

## Timestamp Types

Each evidence pack contains multiple timestamps:

1. **pack_created** — when pack was initialized (auto)
2. **evidence_collected** — when last evidence was gathered
3. **review_started** — when reviewer opened pack
4. **review_completed** — when reviewer made decision
5. **pack_sealed** — when pack was locked for archival

## Ordering Rules

All timestamps MUST satisfy:

```
pack_created < evidence_collected < review_started < review_completed < pack_sealed
```

No timestamp may equal another. No timestamp may be in the future.
No timestamp may be earlier than any predecessor.

## Format

- All timestamps use ISO 8601 with UTC offset
- Precision: milliseconds minimum
- Example: `2026-05-27T14:30:00.000Z`

## Clock Source

- Timestamps generated by system clock at time of event
- No manual timestamp entry — all auto-captured
- Clock skew tolerance: 0 (strict ordering, no buffer)
- If clock skew detected, pack MUST be rejected

## Immutability

- Once sealed, no timestamp may be modified
- Timestamp gaps are acceptable (reviewer may pause)
- Long gaps (>24h) trigger warning but not auto-reject

## Constraints

- No future timestamps
- No backdated timestamps
- No manually entered timestamps
- No timezone conversion after seal
- release_hold = HOLD — timestamps do not grant release
