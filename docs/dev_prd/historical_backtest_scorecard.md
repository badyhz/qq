# Historical Backtest Scorecard — Grading Criteria

## Grade Categories

### PASS
All of:
- `expectancy_r > 0`
- `sample_quality_score >= min_sample_quality_score` (default 0.3)
- No blockers (insufficient candidates, drawdown exceeded)

### WATCH
Any of:
- `expectancy_r <= 0` but no blockers
- `sample_quality_score < min_sample_quality_score` but no blockers
- Mixed signals: positive expectancy but low quality

### REJECT
Any blocker:
- `candidate_count < min_candidate_count` (default 5)
- `max_drawdown_r < max_drawdown_threshold` (default -5.0R)

### INSUFFICIENT_SAMPLE
- `candidate_count < min_candidate_count`
- Treated as REJECT with reason code `insufficient_candidates`

## Quality Gates

| Gate | Threshold | Type | Effect |
|------|-----------|------|--------|
| min_candidate_count | 5 | Hard | REJECT if below |
| min_sample_quality_score | 0.3 | Soft | WATCH if below |
| max_drawdown_r | -5.0R | Hard | REJECT if worse |
| expectancy_r | > 0 | Soft | WATCH if <= 0 |

## Sample Quality Score

Composite score in [0, 1] with three equally weighted components:

1. **Count Score**: `min(candidate_count / 10, 1.0)`
   - Full score at 10+ candidates
   - Linear scaling below 10

2. **Data Completeness**: fraction of outcomes with non-zero MFE and MAE
   - Requires both favorable and adverse excursion data
   - Penalizes missing excursion tracking

3. **Balance Score**: `1 - |win_rate - 0.5| * 2.0`
   - Penalizes extreme win rates (too high or too low)
   - Full score at 50% win rate
   - Zero score at 0% or 100% win rate

## Reason Codes

Generated by `grade_run()`:
- `candidate_count=N<M` — insufficient sample
- `low_sample_quality=X<Y` — below quality threshold
- `non_positive_expectancy=X` — not profitable
- `drawdown_exceeded` — max drawdown too deep

## Experiment-Level Grading

`grade_experiment()` aggregates run grades:
- Verdict is the worst of individual run grades
- REJECT if any run is REJECT
- WATCH if any run is WATCH (and none REJECT)
- PASS only if all runs are PASS

Output includes:
- `experiment_id`, `verdict`
- `run_grades` list with per-run details
- `pass_count`, `watch_count`, `reject_count`
